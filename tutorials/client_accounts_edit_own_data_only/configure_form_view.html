The next step is to configure a View:

  1. Automatically store what Form Tools account created the submission.
  2. Set the view to only show submissions for the current client account.

Which View?

You can read more about Views here, but the important things to know are: you can create as many Views as you want; each View can be configured differently to show different form data; by default, every new Form gets a new View created along with it called "All submissions".

For this tutorial, you can edit any View you'd like - so perhaps just make these changes default All Submissions view. You can rename it or add more Views later once you get a better feel for it.

Editing the View

Tracking show created the submission

Now click on the Views tab within the Edit Form pages. There, edit one of the Views (we'll pick the default "All Submissions" View, as shown in the screenshots).

{% include screenshot.html item="tutorial-client-map-edit-view-general-tab.png" %}
  • On the General tab, set the Access field to public. This will allow all clients that have access to the form also have access to this particular View.
  • Set the Can add submissions to yes.
  • In the Default Values for New Submissions setting, add the field as shown in the screenshot to the right, namely: Field Account ID (the field you just created in the previous step) and default value: {% raw %}{$ACCOUNT_ID}{% endraw %}
    {% raw %}{$ACCOUNT_ID}{% endraw %} is a placeholder. It gets replaced with the user's account ID when the submission is created. For a full list of placeholders, see the section on the Default Values for New Submissions field.
  • Click save.

What you've just done is configure your View to store the account ID of whatever user just created the form through the Form Tools interface. As mentioned before, submissions created from an external, API or Form Builder form won't have this value populated: it's only for submissions created within Form Tools.
